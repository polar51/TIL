#### 0레벨 3번 (복기)

%와 // 를 헷갈림

- 나머지 연산자 ($\%$): 어떤 수를 100으로 나눈 나머지는 항상 $0$부터 $99$ 사이의 값입니다. 이는 곧 원래 수의 가장 오른쪽 두 자리 숫자입니다.
- 정수 나눗셈 연산자 ($\//$): 어떤 수를 100으로 나눈 몫은 원래 수에서 오른쪽 두 자리를 버린 결과와 같습니다.

### node MVC 패턴

#### route

- 기존에 express로 전환한 이유가 메소드에 따른 코드량 증가였는데 routes라는 방식으로 미들웨어로써 공통 api 라우트를 가져갈 수 있고 이 라우트를 통해 원치 않는 라우트로 api 통신을 하는 경우에 404 같은 예외 처리를 할 수 있다는 것과 반복 코드 감소 효과가 있다.
- 예전 첫 취직시에 작성했던 방식은 쿼리문을 직접 문자열 + 변수를 통해 직접 쏘는 방식이었는데 이 방식은 SQL 인젝션과 같은 보안적인 면에서 문제가 있다는 걸 깨달았다. Prepared Statements 또는 Parameterization 기능을 사용하는 것이 보안상 좋을 것으로 예상된다. 이를 도와주는 orm/쿼리빌더 등을 찾아보는게 좋을듯
- node의 패턴도 파악했으니 간단하게 모놀리식 방식으로 타입스크립트, node 패턴 연습을 해볼 토이 프로젝트를 진행해보면 좋을듯
